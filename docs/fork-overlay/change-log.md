# Fork Overlay Change Log

## 记录规则

- 记录的是 overlay 能力定义的变化，而不是普通代码提交流水。
- 如果只是内部重构且行为不变，可以标记为“结构调整，无语义变化”。
- 如果同步 upstream 后能力判断或适配结论发生变化，应同时配套写一份兼容性报告。

## 2026-04-01

### 新增

- 建立 `Fork Overlay` 文档体系，明确区分能力基线、版本变化和同步兼容性报告。
- 新增机器可读账本 `overlay-ledger.json`，用于记录 overlay 版本、能力清单、同步配置和 replay 分层。
- 新增 overlay 重建脚本与账本校验脚本，用于支持 replay-based 同步流程。

### 结构调整，无语义变化

- 将 fork 主干同步流程从“仅依赖代码合并”提升为“代码重建 + 能力校验 + 文档留痕”的治理模式。

### 待后续记录

- 当 Claude / Codex 行为边界发生调整时，在此处补充能力语义变化。
- 当 replay 分层重新定义时，在此处补充 overlay 重放层级变化。
